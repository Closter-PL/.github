# Planning Methodology

> **Important:** This document explains how Closter's planning system works, connecting strategic roadmap to day-to-day execution.

## Overview

Closter uses a four-tier planning system that bridges strategic vision with tactical execution:

1. **Roadmap** (external / strategic)
2. **Milestones** (bridge)
3. **Issues** (real execution)
4. **Weekly Planning** (the key)

## Step 3 â€” Issues (Real Execution)

- Issues don't have fixed dates
- They are prioritized
- They are reordered
- Some are split
- Others are discarded

ðŸ‘‰ **"This is how we work internally."**

Issues represent the actual work items. They live within milestones but maintain flexibility to adapt to changing priorities, technical discoveries, and operational needs.

---

## Step 4 â€” Weekly Planning (The Key)

**This is what's not being visualized elsewhere.**

Each week, the team decides:
- Which issues enter the sprint
- Which ones exit
- Which ones are blocked

**We don't plan the entire month at once because the cost of error is extremely high.**

This is normal practice, not improvisation. Weekly planning allows the team to:
- Respond to technical discoveries
- Adjust priorities based on real progress
- Handle blockers immediately
- Maintain velocity without over-commitment

---

## Why This System Works

1. **Strategic clarity** for external stakeholders (roadmap)
2. **Structural alignment** between strategy and execution (milestones)
3. **Tactical flexibility** for day-to-day work (issues)
4. **Adaptive planning** that reduces error cost (weekly planning)

This methodology ensures that Closter maintains strategic direction while preserving the agility needed for technical execution.
